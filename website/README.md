# Harry McLaren - Personal Website

This is the source code for your static, markdown-powered personal website, designed collaboratively by the Product Manager and UX Designer skills, and built by the Developer skill.

## Core Features
1. **Security First:** Strict Content Security Policy (CSP), no backend databases.
2. **Performance:** Lightweight Vanilla CSS, zero heavy JS framing.
3. **IDE Maintainability:** To add a blog post, you just write Markdown.

## Project Structure
- `assets/` - Contains `styles.css` (the minimalist, monochrome design system with deep teal accents).
- `content/blog/` - Your workspace. This is where you write your `.md` files.
- `templates/` - Jinja2 HTML templates used to structure the pages.
- `public/` - **DO NOT EDIT.** This folder is auto-generated by the build script and contains the final static HTML.
- `build_site.py` - The Python script that parses your markdown and generates your site into `/public`.

## How to Maintain This Site

### Prerequisites
You need Python installed, and a few small packages.
```bash
pip install markdown python-frontmatter jinja2
```

### Writing a New Blog Post
1. Open your IDE (Claude Code, VS Code, etc.).
2. Navigate to `content/blog/`.
3. Create a new file ending in `.md` (e.g., `new-speaking-engagement.md`).
4. At the very top of the file, add Frontmatter metadata:
```markdown
---
title: "Title of your post"
date: "YYYY-MM-DD"
summary: "A short 1-2 sentence description for the blog list page."
---
Write your content here...
```
5. Save the file.

### Building and Publishing
Whenever you change a template, or add/edit a markdown file:
1. Open your terminal in this repository.
2. Run: 
```bash
python build_site.py
```
3. The script will regenerate the `public/` folder.
4. You can preview the changes by opening `public/index.html` in your browser.
5. Deploy the `public/` folder to your static host (e.g., Netlify, Vercel, GitHub Pages, or an AWS S3 bucket).

*Remember: This workflow ensures your site remains lightning-fast and impenetrable compared to traditional WordPress setups.*

## CI/CD Deployment with GitHub Actions & Pages

I have scaffolded a GitHub Actions workflow at `.github/workflows/deploy.yml`. 

**To deploy your site automatically:**
1. Push this repository to GitHub.
2. Go to your repository **Settings** -> **Pages**.
3. Under **Build and deployment** -> **Source**, select **GitHub Actions**.

Now, every time you push a new markdown file to the `main` branch, GitHub Actions will automatically install the Python dependencies, run `build_site.py`, and securely deploy the output to GitHub Pages!
